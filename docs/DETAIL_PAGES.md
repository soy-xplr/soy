# 북마크 세부 페이지 요구사항

## 목적

북마크 카드는 목록에서 요약 정보를 보여주고, 세부 페이지는 사이트 소유자가 직접 작성한 텍스트와 이미지를 통해 더 깊은 큐레이션 맥락을 전달한다.

## 라우팅

- 각 링크 데이터는 `slug`를 가진다.
- 카드 클릭 시 `/bookmarks/{slug}` 형태의 내부 세부 페이지로 이동한다.
- 세부 페이지에서 원본 외부 링크는 새 탭으로 연다.

## 콘텐츠 작성 방식

- 첫 버전에서는 세부 페이지 본문을 데이터 파일, CMS, 마크다운으로 관리하지 않는다.
- 각 세부 페이지의 텍스트와 이미지는 페이지 코드 또는 컴포넌트 안에 직접 하드코딩한다.
- 링크 데이터는 목록과 라우팅에 필요한 요약 정보만 가진다.

## 세부 페이지 기본 구성

1. 목록으로 돌아가기 링크
2. 제목
3. 카테고리, 태그, 저장일
4. 대표 이미지 또는 커버 이미지
5. 직접 작성한 본문 텍스트
6. 본문 중간 이미지
7. 개인 메모 블록
8. 원본 링크 열기

## 권장 본문 예시

```txt
이 링크를 저장한 이유

처음 봤을 때 좋았던 점

나중에 다시 참고하고 싶은 부분

비슷한 분위기의 다른 링크와 연결되는 생각
```

## 완료 기준

- 모든 북마크 카드가 내부 세부 페이지로 이동한다.
- 세부 페이지는 노션 일반 페이지처럼 텍스트와 이미지를 세로로 읽는 구조다.
- 세부 페이지 본문은 직접 수정하기 쉬운 위치에 하드코딩되어 있다.
- 원본 링크는 세부 페이지에서 새 탭으로 열린다.
